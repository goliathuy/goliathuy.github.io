🚀 Project Overview: Dual-Purpose GitHub Pages Site

A dual-purpose GitHub Pages site combining a professional developer hub with a Kegel exercise timer, built as an exploration of AI-assisted development with minimal direct code manipulation.

Current timer app (primary): /kegel/ — Vite + React in kegel-ui/, built to kegel/. The legacy kegel-timer.html page is a short redirect bridge to the same origin.

Local Development Setup

  1. Recommended: Use start-timer.bat to launch Python HTTP server at localhost:8000
  2. Alternative: Open HTML files directly (service workers won't function)
  3. Testing: Open tests/test-runner.html for custom test framework
  4. Debugging: Use VS Code launch configurations for Chrome debugging

Available VS Code Tasks

• Start Timer Server: Launches Python development server • Start Test Server: Alternative HTTP server for testing

🎯 Core Implementation Patterns

JavaScript Architecture

  • No bundling: Direct script includes, vanilla JavaScript for maximum compatibility
  • DOM-first approach: Heavy use of getElementById() and direct DOM manipulation
  • Defensive coding: Extensive null checks prevent runtime errors in AI-generated code

Timer Implementation

// Core timer pattern setInterval(() => { if (countdown <= 0) { // Phase transition logic playSound(isHoldPhase); vibrate(200); updateTimerVisuals(); } countdown--; }, 1000);

PWA Service Worker Pattern

// Cache-first strategy with fallback self.addEventListener('fetch', (event) => { event.respondWith( caches.match(event.request) .then(response => response || fetch(event.request)) ); });

🚨 Development Notes

AI Development Context: Comments throughout indicate AI-assisted development. The codebase emphasizes defensive programming patterns essential for AI-generated code robustness.

Common Development Tasks

Adding New Exercise Routines

Modify the exercise objects in script.js and add corresponding UI buttons in kegel-timer.html.

Styling Changes

Edit styles.css - changes affect both applications. Use class naming that indicates which app (e.g., .timer-specific vs .hub-specific).

PWA Updates

Update CACHE_NAME in sw.js and add new assets to ASSETS_TO_CACHE array when adding files.
